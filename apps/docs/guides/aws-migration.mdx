# AWS Migration

AgentDomain now targets Aurora/RDS PostgreSQL. Keep the Drizzle schema unchanged and cut over by changing `DATABASE_URL` after data validation.

## Database

1. Create an Aurora PostgreSQL or RDS PostgreSQL cluster in private subnets.
2. Put the app and workers in the same VPC, preferably ECS Fargate behind an ALB.
3. Use RDS Proxy for pooled private database access.
4. Use AWS DMS full load plus CDC from Neon into AWS Postgres.
5. Run `pnpm --filter @agentdomain/web db:migrate` against the AWS URL.
6. Compare table counts, run preflight, then switch production `DATABASE_URL`.
